# 文笔技法 (Writing Style)

> Generator/Fixer 写作时的句、段、词、描写四项技术规范。
> 本文件**不**包含人物心理学、剧情逻辑、AI 味判据——
> 那些分别在 `rules/writing-iron-laws.md` / `rules/iron-laws.md` /
> `rules/ai-rhythm-taboos.md`。

## 一、句式技法

- 长短句结合：不可连续 3 句以上相同长度的句子；长句承载信息，短句制造节奏感。
- 少用「了」字：连续「了」字产生流水账感——「他走了过去，看了一眼，又走了回来」→ 改成「他走过去，扫一眼，折返。」
- 转折词使用上限：「虽然……但是」「然而」「却」「若……则」「因为……所以」一段 ≤ 1 次，用多了像在强迫读者同意你。
- 动词/名词多于形容词：用精准的动作和事物本身推动画面，而非堆砌「华丽的」「优雅的」「深邃的」。

## 二、段落技法

- 每段 1 个核心信息点：一段话只讲一件事，读者扫一眼就抓到主旨。
- 每段 3-5 行（移动端阅读）：手机屏一屏能看完一段为宜，避免密密麻麻。
- 第一句承上、最后一句启下：段落之间靠首尾句衔接，不靠转折词硬接。

## 三、描写技法

- 五感用具象「名词+动词」，不堆形容词；具体五感示例与「非五感反例」见 `rules/writing-iron-laws.md` iron_law_18。
- 比喻精准化、点睛使用：比喻必须有画面感，「那双眼睛像饿狼见了肉。」具体使用上限阈值（明喻每千字 ≤ 8）见 `rules/ai-rhythm-taboos.md`。
- **标签化开篇**：用 100 字内交代「谁、在哪、正在经历什么」，快速贴标签。
  > 例：「小爷我乃镇南府世子林峰，不过是区区六百两酒钱，醉红楼我欠了三千两都不敢这么对我，还不快给小爷我松绑？」——一句交代身份、地位、性格（纨绔但有底气）、冲突场景。
- **具体化场景**：描述必须落到读者能在脑海中「看见」的具体事物，用日常即景让读者自动联想。
  > 例：不写「这是一个压抑的下午」，而写「夕阳下教学楼后的小树林，少年和少女相对而站，谁都不先开口。」
- 五感示例：视觉「夕阳的火烧云把半边天染成铁锈色」/ 听觉「雨水砸在铁皮瓦上，像有人往屋顶扔石子」/ 嗅觉「大雨过后泥土的腥味混合着街边大排档的镬气」/ 触觉「潮湿的短袖黏在后背上」/ 味觉「冻柠茶的甜和涩在舌根打架」。

## 四、人称与视角

- 叙事人称统一：确定第三人称「他/她」或第一人称「我」后全书一致，不可混用。
- 多人对话场景主要发言者 ≤ 3 人：超过 3 人轮番发言会让读者迷失，必要时用「沉默的旁观者」消化人数。
- 适度口语化和歇后语：符合人物身份的市井语言——「这是把脑袋别在裤腰带上讨生活。」
- 开篇精简：首三章场景 ≤ 3 个、出场人物 ≤ 3 个，通过事件中的行动立人设，不要逐个介绍。
